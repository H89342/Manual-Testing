# 02 — Analysis & Q&A

Store requirement analysis documents and Q&A checklists here.

## What goes here:
- Analysis output from `/02-analyze` → `analysis-[feature-name].md`
- Q&A checklists from `/03-qa` → `qa-[feature-name].md`
- Confirmed answers and assumption logs

## Naming convention:
- `analysis-[feature-name].md`
- `qa-[feature-name].md`

## Status tracking:
Each Q&A file should have a status header: `OPEN | IN REVIEW | RESOLVED`

## Next step:
When all CRITICAL Q&A questions are resolved → run `/04-create-tc`
