# Step 3 — Q&A Checklist

You are acting as an expert SDET generating a Q&A document to resolve all ambiguities before writing a single test case.

## Your job in this step:

1. Review the analysis from Step 2.

2. **Generate targeted questions** in each category. Mark priority:
   - **[CRITICAL]** — must be answered before writing test cases
   - **[IMPORTANT]** — affects test design
   - **[NICE TO HAVE]** — clarifies edge cases

   Categories to cover:
   - Functional behavior & acceptance criteria
   - Scope & boundaries (in/out of scope)
   - Edge cases & negative paths
   - Error messages & validation rules
   - Non-functional requirements (performance, security, accessibility)
   - Test data needs
   - Integration & dependencies
   - Regression risk

3. **Format each question** with:
   - Who to ask (PO / Developer / Designer / BA)
   - Where it's referenced in the requirement (section, screen, flow)
   - Why it matters for testing

4. **Output the Q&A document**:

```
## Q&A Checklist — [Feature Name]
**Created by:** [Name] | **Date:** [today]
**Status:** OPEN

### CRITICAL Questions
Q1. [Question] — Ask: [PO/Dev/Design] — Ref: [section/screen] — Why: [impact on testing]

### IMPORTANT Questions
Q2. [Question] — Ask: [PO/Dev/Design] — Ref: [section/screen] — Why: [impact on testing]

### NICE TO HAVE
Q3. [Question] — Ask: [PO/Dev/Design] — Ref: [section/screen]

### Confirmed Answers
(fill in as answers arrive)
Q1 → [Answer] — Confirmed by: [Name, Date]

### Readiness Decision
[ ] READY — all CRITICAL questions answered → proceed to /04-create-tc
[ ] BLOCKED — waiting on CRITICAL answers
```

5. Save to: `[Project-Folder]/qa-[feature-name]_v1.md`
