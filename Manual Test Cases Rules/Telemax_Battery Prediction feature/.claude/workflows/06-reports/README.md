# 06 — Reports

Store all test execution summary reports and sign-off documents here.

## What goes here:
- Test Execution Summary Reports produced at end of each cycle
- Sign-off documents
- Metrics snapshots (pass rate, defect density, coverage %)

## Naming convention:
`report-[feature-name]-[YYYY-MM-DD].md`

## Minimum content per report:
- Build version and environment tested
- Execution results table (Pass / Fail / Blocked / Skipped)
- Defect summary by severity
- Outstanding risks
- Sign-off recommendation: PASS / CONDITIONAL PASS / FAIL
